## Add User

You can add other user accounts to the workspace. Before you add a new user, you should define its role, i.e., select one of the predefined roles or create a new one. Then, click Add User in the Users tab and fill out the form shown below.


To create a new user, you should:

* Go to the **Users** tab;

* Click the **Add User** button on the toolbar of the server.

**User Menu**

In this menu, you can specify all user information and manage the user account.


![](../../images/new_user.png)


![](../../images/img_1.png) This field contains the first name.

![](../../images/img_2.png) This field contains the last name.

![](../../images/img_3.png) This field contains the email that will be used as user login and for authentication.

![](../../images/img_4.png) In this field you should insert a password to protect your account from unauthorized access. Keep in mind - the combined and complex password (consisting of letters and numbers) enhances account security and reduces the risk of loss of sensitive data.

![](../../images/img_5.png) This field specifies the role for the user account.

![](../../images/img_6.png) Activate or Deactivate the user. The account will be activated if the checkbox is set to true. The account will be deactivated if the checkbox is set to false, and this account cannot be used to login to the server workspace.

![](../../images/img_7.png) This field specifies the [root folder](#RootFolder) for the user account, the root folder for the server components in the current workspace.

![](../../images/img_8.png) In the picture field, you can upload an image that will be the user's avatar.

**Root Folder**

When you create or edit a user account, you can specify the root folder for it. This folder will be the beginning of the hierarchy of server items for the current user account. Below is a diagram of components for the root administrator:


![](../../images/root_folder.png)


For example, for a certain user account, we specify **Folder4** as the root. Then when logging into the server workspace, **Folder4** will be the root directory for this user account, and the hierarchy of server elements will be as follows.


![](../../images/root_folder_1.png)


All elements in the workspace located in folders of a higher level will not be displayed to the current user.
